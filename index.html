Clinical Intake & Documentation Agent

Built a 6-step clinical intake pipeline: ingest → PHI redaction → LLM extraction → validation → summarization → FHIR export. Orchestrated via async Celery workers with provider-agnostic LLM layer (Claude + Llama interchangeable behind one interface). Requirements gathered from a regulatory affairs professional. React review UI with per-field confidence scoring routes low-confidence extractions to human review.

Built eval harness with labeled synthetic test set and LLM-as-judge graders. Llama 3.1 8B scored 90.6% extraction / 0% hallucination on extraction — but the harness surfaced a hallucination in the summarization stage (model inferred patient demographics not in source text), a gap the extraction-only suite didn't cover. Enforced via pytest regression suite running on every CI commit.
